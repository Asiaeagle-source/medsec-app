## §9 自驗鐵律(2026-07-19 起生效)

CC 每次 push 後、回報 Lynn 前,**必須先自行驗收**。
無自驗證據的「已完成/已修復,請驗收」**視同未完成,不接受**。

### 一、環境設置(首次執行,之後沿用)
1. 安裝 Playwright:`npm i -D playwright && npx playwright install chromium`
   - ☁️ 雲端容器註記:本開發容器已預裝 Playwright(`/opt/node22/lib/node_modules/playwright`)
     與 Chromium(`/opt/pw-browsers/`),**免重複安裝、勿跑 `playwright install`**,
     launch 時指定 `executablePath` 即可。容器回收後映像檔仍在,沿用。
2. 驗證環境可用:最小腳本開啟本機 dev server 首頁、截圖、印出頁面 title,確認能跑。
3. 若 Vercel preview 有存取保護打不進去,改用本機 dev server。
   - 本 repo 為靜態站(無 `npm run dev`):`python3 -m http.server <port>` 等效。
4. 測試帳號固定用 **0007**(密碼依 AE+工號慣例),**嚴禁用其他帳號**。

### 二、每次自驗必做
1. 用 0007 實際操作本次改動的完整流程:登入 → 進入頁面 → 執行操作 → 檢查結果。
2. 斷言三件事:
   a. 畫面出現**預期文字/元素**(不是「沒報錯」就算過)。
   b. Console **零紅字**(warning 要列出並說明)。
   c. 涉及寫入的操作:寫入後 **SELECT 回來核對**,筆數與欄位值符合預期
      (error ≠ 零筆,error 必須浮出)。
3. 迴歸快檢:同頁面的既有功能點過一輪,確認沒被改壞。

### 三、測試資料紀律
1. 自驗建立的資料,可辨識欄位(備註/說明)一律加前綴 **[TEST]**。
2. 自驗結束立即清理:刪除本次建立的所有測試列,並回報清理 SQL 的執行結果
   (**交 Lynn 執行,CC 不跑 SQL**)。
3. 不可動到 0007 以外任何真實使用者的資料。

### 四、回報格式(每個 PR / 每次修復)
1. 自驗步驟清單:每步「操作 → 預期 → 實測」三欄。
2. 關鍵畫面截圖(至少:操作前、操作後)。
3. Console 輸出摘要。
4. 寫入類操作的 SELECT 核對結果。
5. 最後一行寫:「**自驗通過,待 Lynn 驗收**」。

### 五、與既有規矩的關係
- §8 二振條款照舊:自驗過但 Lynn 驗收仍失敗兩次,第三輪先診斷後修復。
- CC 永不跑 SQL、永不 merge,不變。
- 自驗不取代 Lynn 驗收:Lynn 驗業務邏輯,自驗只保證「功能沒壞」這條底線。
- 純後端變更(如 cron)無 UI 流程可操作時,自驗 = headless 契約/整合測試
  + 真環境執行證據(回傳 JSON / DB 抽查),其餘條款(證據先行、[TEST] 紀律)照用。
